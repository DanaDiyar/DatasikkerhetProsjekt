Hva er gjort?

1. Opprettelse av server med Apache

Apache installeres for å håndtere forespørsler. Etter installasjonen må brannmuren konfigureres for å tillate trafikk. Deretter startes og aktiveres Apache, og status kan sjekkes for å bekrefte at serveren kjører. For å teste om serveren fungerer, åpnes IP-adressen i en nettleser.

2. PHP-installasjon og konfigurasjon

PHP og nødvendige moduler installeres. For å bekrefte at PHP fungerer, opprettes en testfil i serverens rotmappe som deretter åpnes i nettleseren.

3. Oppsett av MySQL database

MySQL installeres for å håndtere datalagring. Etter installasjonen sikres MySQL-konfigurasjonen ved å sette passord og aktivere nødvendige sikkerhetsinnstillinger. Deretter logges det inn i MySQL, og databasen opprettes sammen med nødvendige tabeller for meldinger og brukere.

4. Konfigurering av Apache for prosjektet

Apache må konfigureres for å vite hvor prosjektfilene ligger. En ny konfigurasjonsfil opprettes, som peker til prosjektets mappe. Denne aktiveres, og Apache restartes for å laste inn endringene.

5. Plassering av filer

Prosjektfilene ligger organisert i en bestemt mappe på serveren. Denne inneholder blant annet API-filen som håndterer forespørsler, en databasekoblingsfil som muliggjør tilkobling til MySQL, og en indeksfil for hovedsiden. Databasekoblingsfilen definerer servernavn, brukernavn, passord og databasenavn. Deretter opprettes en tilkobling til databasen, og hvis denne feiler, vises en feilmelding. API-filen håndterer forespørsler til serveren. Ved en GET-forespørsel kan den hente meldinger fra databasen eller rapportere en melding som upassende. Hvis handlingen er ugyldig eller mangler, returneres en feilmelding. Til slutt lukkes tilkoblingen til databasen.

6. URL-basert API

API-et testes ved å sende en forespørsel til serveren og sjekke responsen. Hvis alt fungerer, skal URL-en sende deg til ønskelig side

7. Feilsøking

Dersom noe ikke fungerer som forventet, kan følgende tiltak gjøres: Sjekke Apache-loggene for eventuelle feil. Verifisere at MySQL kjører som forventet. Kontrollere at filene har riktige rettigheter og eier.
